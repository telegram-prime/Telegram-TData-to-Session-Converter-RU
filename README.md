# TD2S Converter – Telegram **TDATA → SESSION** (или SESSION+JSON)  
Инструмент, который преобразует аккаунты в формате TDATA (Telegram Desktop) в файлы сессий Telegram (SESSION или SESSION+JSON).

---

## ⚙️ Основные функции

| Функция | Описание |
|---------|----------|
| **TDATA → SESSION / SESSION+JSON** | Переводит TDATA‑аккаунты Telegram Desktop в формат SESSION или SESSION+JSON. |
| **Многопоточность** | Параллельная обработка нескольких файлов, ускоряя массовое преобразование. |
| **Конвертация без подключения** | Возможность преобразовать SESSION+JSON в TDATA без онлайн‑соединения с аккаунтом. |
| **Совместимость со старым лаером** | Опция сохранения совместимости со старой версией Telegram лаера или запись как новый слой. |
| **Создание новой сессии** | Возможность создать новую сессию для аккаунта либо сохранить текущую. |
| **TDATA‑реавторизация** | Используется как функция реавторизации TDATA (экспорт TDATA). |
| **Пользовательские попытки / задержка** | Настройка количества попыток подключения и паузы между ними. |
| **Отключение других сессий** | При необходимости можно отключить все остальные активные сессии аккаунта во время преобразования или переавторизации. |
| **Конфигурация эмуляции устройств** | Гибкая настройка устройства, используемого для подключения к сессии. |
| **Поддержка прокси** | Для каждой соединения можно задать уникальный HTTP или SOCKS5 прокси (поддержка большого количества прокси). |
| **Пропуск недействительных сессий** | Если подключение невозможно – просто пропускается. |
| **Сортировка результатов** | Действительные аккаунты → «valid», недействительные → «invalid» папки. |
| **Журналирование** | Возможность сохранять лог работы. |
| **Резервное копирование** | Создание резервной копии исходных файлов перед конвертацией. |
